Occupancy Grid Map Generation

Implementation

Use lidar scanner data from a moving vehicle to create an occupancy grid map. The process involves implementing the inverse scanner measurement model to extract occupancy probabilities and iteratively updating a belief map through log-odds calculations. This approach constructs a probabilistic occupancy grid that converges to the true map as more data is gathered.

Planning

Implementation

There are 4 components including Path genereation, Path collision checking, Velocity profile generetion and Behavioural planning. With these pipelines, the system was simulated by CARLA at the case of intersection.

  • Camera Calibration: Calibrate the top-mounted camera to the robot arm's coordinate system, establishing a precise spatial relationship between the camera's view and the robot's workspace.
  • Image Processing: Implement image filtering techniques to detect and locate colored boxes within the camera's field of view.
  • Hand-Eye Coordination: Utilize the calibrated system and image processing results to enable the robot arm to accurately pick and place the detected boxes within the workspace.
